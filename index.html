Характеристика

Конкретный вариант: цвет + размер. Однозначно уточняет позицию до единственного SKU.

  • Цвет — Petrol, антрацит, Latte, Grey…
  • Размер — S, M, L/XL, 2XL, 3XL, 4XL…
Petrol, S антрацит, 2XL
Уровень 2-Б
Поле «Артикул»

Отдельный реквизит карточки варианта товара. Артикул не включается в название и не считается характеристикой; он хранится отдельно только как служебный код.

  • Уникален — один артикул = одна позиция+цвет+размер
  • Удобен для поиска — фильтры, ЧЗ, отчёты
1175/79 431/10 603/27
Уровень 2-В
Штрихкод(ы)

Один или несколько штрихкодов на характеристику. По любому — однозначная идентификация SKU.

  • Несколько на одну хар-ку — старый ШК + новый ЧЗ код
  • Сверка заказа — сканирование → 100% идентификация
ЧЗ-код Старый ШК
Правило уникальности — ключ к устранению задвоений

Комбинация «Позиция + Характеристика (цвет + размер)» однозначно идентифицирует конкретный SKU. Любой штрихкод ведёт в одну карточку — задвоений нет. Артикул не входит в состав характеристики и используется отдельно как дополнительный реквизит.

Открытые вопросы
Ответы на ключевые вопросы по номенклатуре
Вопрос 1 ✓ Рекомендация
Нужен ли артикул в составе характеристики?
Нет, не нужен. Характеристика = только «Цвет, Размер» (например, Petrol, S). Артикул не включается ни в наименование, ни в набор характеристик; его лучше хранить отдельно как реквизит для поиска, отчётов и сопоставления.

Что теряем: только визуальное дублирование — ничего функционального.
Вопрос 2 ≈ Уточнить у бухгалтерии
В БП: выгружать товары с характеристиками или без?
• Нужен только общий остаток → без характеристик (одна карточка на модель).

• Нужна аналитика по SKU → с характеристиками.

ВОПРОС: нужна ли детализация по цвету/размеру или достаточно уровня «модель»?
